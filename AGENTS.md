# AGENTS.md — One in Eight Billion

このファイルは、本アプリを開発する **Codex（開発エージェント）** 向けの作業指示書。

体制: **開発＝Codex／監査＝Claude Code**。Codex が実装し、Claude Code が別の第三者として
正しさ・データ・誠実さを点検する。Codex は実装の主、Claude は「盛っていないか」を見る目。

---

## 1. これは何か（ぶれさせない核）

- 年収を入れると、世界 約81億人の中での収入順位／上位%を返す Web アプリ。
- 建前はジョークツール。本意は「**金がないと言う先進国の我々が、実は世界でいかに上位にいるか**」を体感させること。
- したがって **数字の誠実さがこのアプリの商品価値そのもの**。盛った数字は存在意義を壊す。
  「だいたい正しく、嘘はつかない」を死守する。

## 2. 現状（v1）

- 単一ファイル `index.html`（ビルド不要・依存なし）。ブラウザで開けば動く。
- 計算: 入力年収を **PPP（購買力平価）** で international$ に換算 → 世界の1人あたり所得分布モデルに当てる。
  - 本体 = 対数正規分布（定数 `MEDIAN` / `SIGMA` / `MU` / `Z99`）。
  - トップ1%超 = パレート裾（`SPLICE_X` / `TAIL_ALPHA`）＝「ファクトが尽きる領域」のフェルミ推定。
- モデル定数は **すべて `index.html` 冒頭の1ブロックに隔離**（`POP` / `MEDIAN` / `SIGMA` / `Z99` /
  `SPLICE_X` / `TAIL_ALPHA` / `PPP{}`）。主要関数は `cdf()` / `normCdf()` / `erf()` / `reframe()` / `compute()`。
- 背景・方針は README.md。

## 3. 既知の弱点（v1時点・監査で確認済み）

信用度は「**方向・桁＝OK／正確な数字＝まだ**」。直す優先順（影響大きい順）:

1. **単位のズレ（最大）**: 個人の年収を「1人あたり所得の分布」にぶつけている。本来は
   **世帯収入 ÷ 世帯人数** で per-capita 同士を比べるべき。今の作りは全員を実際より上位に盛る。
2. **アンカーが目分量**: `MEDIAN=3000`、トップ1%≈$55k 等は出典なしの概算。World Bank PIP の実値へ。
3. 税引き前/後の不一致、`PPP{}` 係数も概算。
4. パレート `TAIL_ALPHA=2` は当て推量。最上位ほど数字が柔らかい。

## 4. ロードマップ（この順で行く）

- **(A) 単位を揃える**: 世帯人数の入力を足し、per-capita 同士で比較する。← 精度が最も上がる。
- **(B) 実データ較正**: World Bank PIP の分位点を数点焼き込み、本体を「モデル」から「実データ参照（補間）」へ。
  パレートは最上位の裾だけに残す。出典・年・PPP基準をコメント/READMEに明記。
- **(C) 検証**: Giving What We Can "How Rich Am I" の公表値と、同じ入力で突き合わせる。
- 後続: 名目（為替）併記トグル／税引き後・世帯対応／共有用の見せ方（OG画像・一言コピー）。

## 5. 開発の約束（Codex はこれを守る）

- **ファクトには出典を必ず付ける。** 「事実」と称する数字は、出所（データセット名・年・PPP/名目の別）を
  コードコメントか README に残す。**出典のない数字をファクトとして扱わない。**
- **推定（フェルミ）は「推定」と分かる形にする。** UI の "How is this calculated?" は、実装の実態と常に一致させる。
  コードは対数正規なのに説明文が別物、は禁止。**精度を勝手に盛らない**（小数の見せ方で実態以上に正確そうに見せない）。
- **国際比較は PPP**（為替ではなく購買力平価）。名目を出すときは「別物」として明示する。
- 定数は冒頭ブロックへの隔離を維持。マジックナンバーを関数内に散らさない。
- 当面は **単一ファイル・ビルドなし** を維持（分割が必要になったら、理由を添えてから）。
- 大きい/破壊的な変更の前に、**何を・なぜ変えるかを1行残す**（監査が追えるように）。

## 6. 監査（Claude Code）が見る観点 = 完了の定義

Codex の変更は、以下を満たして初めて「完了」とみなす:

- [ ] ファクト数字に出典がある。
- [ ] UI の計算説明とコードの実装が一致している（盛っていない）。
- [ ] 既知の基準と矛盾しない（例: 米 $60k ≒ 世界トップ約1%）。
- [ ] `index.html` を開くだけで動く（依存・ビルド工程を増やしていない）。
- [ ] 変更点と、触った前提（assumption）を1行で説明できる。

## 7. 動かし方・確認

- 実行: `index.html` をブラウザで開く（または `open index.html`）。
- 当たり値（大きく外れたら定数か単位を疑う）: 米 $60k → トップ約0.8〜1%、日本 ¥5M → トップ数%。
